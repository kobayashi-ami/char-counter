# 文字数カウンター

テキストを**カーソルで選択するだけ**で、マウスの近くに文字数がポップアップ表示されるmacOSメニューバー常駐アプリ。

コピー&ペースト不要。メモ、Safari、Pages、Claudeデスクトップなど、どのアプリでも動きます。

```
テキストを選択 →  ┌──────────────────────┐
                │ 42字（空白除く 40）      │
                └──────────────────────┘
```

## 特徴

- 選択するだけ。選択を解除すれば消える
- クリップボードには一切触らない（アクセシビリティAPIで選択テキストを読む）
- 絵文字も1文字としてカウント（書記素単位）
- 空白・改行を除いた文字数も同時に表示
- Electron製アプリ（Claudeデスクトップ等）にも対応
- Liquid Glass風の見た目：背後を透かす本物のブラー、光の反射、出現時に光が横切るアニメーション。ライト/ダークモード自動適応

## ビルドと起動

macOSのCommand Line Toolsがあればビルドできます（Xcode不要）。

```bash
./build.sh
open 文字数カウンター.app
```

## 初回設定（必須）

アクセシビリティ権限がないと選択テキストを読めません。

1. 初回起動時に出るダイアログに従うか、**システム設定 → プライバシーとセキュリティ → アクセシビリティ** を開く
2. 「文字数カウンター」をONにする
3. 反応しない場合は、メニューバーの「字」→ 終了 してから再起動

Mac起動時に自動で立ち上げたい場合は、システム設定 → 一般 → ログイン項目 に追加してください。

> **注意**: 再ビルドすると署名が変わるため、アクセシビリティ権限が無効になります（ONに見えていても効いていない状態）。再ビルド後はスイッチを一度OFF→ONし直すか、`tccutil reset Accessibility com.ami.selectcount` で登録を消してからやり直してください。

## 仕組み

0.2秒ごとにアクセシビリティAPI（`AXSelectedText`、非対応アプリには `AXSelectedTextRange` + `AXStringForRange` のフォールバック）でフォーカス中の選択テキストを取得し、マウス位置の近くにフローティングパネルを表示します。前面アプリが切り替わったときに `AXManualAccessibility` を立てることで、Electron系アプリのアクセシビリティツリーも読めるようにしています。

アクセシビリティ情報を公開していないアプリでは表示されません。

## iPhone版（ショートカット）

iOSでは他アプリの選択テキストを常時監視するAPIが開放されていないため、iPhone版は**共有シートから使うショートカット**として提供しています。

> テキストを選択 → 共有 → 文字数カウンター → 「42字（空白除く 40）」

- `ios-shortcut/文字数カウンター.shortcut` — 署名済み。AirDropでiPhoneに送って開くか、Macのショートカットアプリで開けばiCloud経由でiPhoneに同期されます
- `ios-shortcut/文字数カウンター.shortcut.plist` — ソース。編集したら `plutil -convert binary1` → `shortcuts sign --mode anyone` で再署名
- 選択なしでホーム画面や背面タップから起動すると、クリップボードの文字数を数えます

## ファイル構成

| ファイル | 内容 |
|---|---|
| `main.m` | ソース（Objective-C、1ファイル） |
| `Info.plist` | アプリバンドル設定（`LSUIElement` でDockに出さない） |
| `build.sh` | clangでビルドして .app を作る |
| `ios-shortcut/` | iPhone版ショートカット（署名済み + ソースplist） |

## 終了方法

メニューバーの「字」アイコン → 終了
